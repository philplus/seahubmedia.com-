# GMV1 — 电商数据分析系统架构（v0.1）

> 目标：多主体多店铺电商+广告数据每日自动拉取、入库、分析、生成 WhatsApp 日报（09:00 WIB）。

## 1. 需求口径（已确认）
- 报告时间：每日 09:00 WIB
- 统计口径：订单“下单创建”（order created）
- GMV：净销售额（扣除退货），平台口径
- 多主体/多店铺：每店输出 Top10 商品
- 历史回拉：近 3 个月
- 运行环境：Mac mini

## 2. 数据源（分层接入）
### P0：TikTok Shop（订单/退款/商品）
- 拉取：订单列表（含创建时间/状态/金额/币种）、退款/退货、商品维度销量。
- 输出：按日、按店聚合 + Top10 商品。

### P1：广告（TikTok Ads / Meta / Google 视你接入优先级）
- 拉取：消耗、展示、点击、CTR、CPC、转化、ROAS（或转换事件）。

### P2：运营辅助数据
- 活动日历、价格表/成本表、库存（如有）。

## 3. 系统模块（Pipeline）
1) **Ingest 拉取层**
- 每个数据源一个 connector（可独立运行/重试）。

2) **Normalize 规范化层**
- 把不同源字段映射到统一 schema；统一时区（WIB）与币种字段。

3) **Store 存储层（SQLite v0）**
- 本机 sqlite 文件，支持增量写入、幂等 upsert。

4) **Compute 指标计算层**
- 生成日表：orders_daily / returns_daily / products_daily / ads_daily。
- 计算 DoD 变化、Top10。

5) **Detect 异常检测层**
- 规则 + 阈值（先静态，后可按店铺动态）。

6) **Report 报表生成层**
- WhatsApp 文本模板（短、可行动）。

7) **Deliver 投递层**
- OpenClaw message → WhatsApp（你 + 工作汇报联系人）。

8) **Audit 日志层**
- 每次运行写入 runs 表（开始/结束/耗时/错误/数据量）。

## 4. 运行策略
- **回拉任务（一次性）**：拉取 3 个月历史 → 入库 → 生成基线。
- **每日任务**：
  - 08:30 拉数（可分源并发）
  - 08:40 入库 + 聚合 + 异常
  - 09:00 生成并发送日报

## 5. 安全与权限
- Token/Secret：只存本机（环境变量/本地密钥文件），不发到聊天。
- 不做自动下单/自动改价等高风险写操作。
